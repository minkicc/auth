MKAuth / OIDC / PKCE

Minimal public-client login, without hidden magic.

This page demonstrates the standard Authorization Code + PKCE flow against the OIDC endpoints exposed by MKAuth. It performs discovery, sends the browser to /oauth2/authorize, exchanges the code at /oauth2/token, reads /oauth2/userinfo, and supports browser-session logout through /oauth2/logout.

Quickstart defaults: issuer http://127.0.0.1:8080, client demo-spa, redirect URI http://127.0.0.1:3000/.

What this proves Your current checkout exposes a standards-based browser login path.
What it does not do It does not hide tokens behind a BFF. This is a public-client demo on purpose.
Best next production step For web apps with a backend, prefer a BFF and keep tokens off the browser when possible.
